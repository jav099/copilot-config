## Context

This focus area provides deep knowledge specific to the CSS Gap Decorations feature in Blink.
It supplements (not replaces) the base repo knowledge in `../sections/` and the `css` and `layout` focus areas.

The implementation has three pillars: **Style**, **Layout**, and **Paint**. Layout and Paint are the primary areas of work. Gap decorations are currently implemented for **grid**, **flex**, and **multicol** containers. **Grid-lanes** (masonry) is the next container type to implement.

**Use this when:**

- Working with `GapGeometry` construction in layout algorithms (grid, flex, multicol, grid-lanes)
- Debugging or extending gap decoration painting (`GapDecorationsPainter`, `BoxFragmentPainter`)
- Understanding the Main-Cross (MC) geometry model and intersection computation (see `layout/gap/README.md`)
- Adding or modifying gap decoration CSS properties (rule-width, rule-style, rule-color, etc.)
- Working with `GapDataList` / `GapData` style data structures
- Handling gap decoration paint invalidation
- Implementing fragmentation support for gap decorations
- Consulting the CSS spec at `/Users/javiercon/spec/javier-gaps-1/css-gaps-1/`
